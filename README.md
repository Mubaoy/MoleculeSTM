# MoleculeSTM under ChemVL Protocol

This repository is a fork of [chao1224/MoleculeSTM](https://github.com/chao1224/MoleculeSTM) with an added ChemVL-style evaluation package.

The purpose of this fork is to reproduce MoleculeSTM molecular property prediction results under the same protocol used by ChemVL-style baselines: the same MoleculeNet datasets, split rules, run seeds, metrics, and compact result format. The original MoleculeSTM implementation is kept as the model backbone; the added code only wraps it for ChemVL-compatible evaluation.

## Project Scope

Supported molecule branches:

| Branch | Input | Role in this fork |
|---|---|---|
| Graph | Molecular graph | MoleculeSTM graph encoder fine-tuned on ChemVL MoleculeNet splits |
| SMILES | SMILES sequence | MoleculeSTM MegaMolBART/SMILES encoder fine-tuned on the same splits |

Benchmark tables:

| Table | Split | Datasets |
|---|---|---|
| A | `scaffold` | BACE, BBBP, ClinTox, HIV, SIDER, Tox21, ESOL, FreeSolv, Lipo, QM7 |
| B | `random_scaffold` | BACE, BBBP, ClinTox, HIV, SIDER, Tox21, ESOL, FreeSolv, Lipo, QM7 |
| C | MoleculeACE MolMCL protocol | Configs/splits are included; MoleculeSTM Table C runs are not completed in this fork |

Metric convention follows ChemVL: classification uses ROC-AUC, QM7 uses MAE, and ESOL/FreeSolv/Lipo use RMSE. Each `(branch, dataset, split)` setting is run with `runseed = 1, 2, 3`.

## Repository Layout

```text
chemvl_benchmark/
  README.md                 Detailed benchmark notes
  DECISIONS.md              Reproducibility decisions and implementation notes
  SUBMIT_FILES.md           Submission checklist
  configs/                  ChemVL-style task configs
  splits/                   Fixed train/valid/test split indices
  scripts/                  Run, audit, and summarization scripts
  results/                  Compact committed result summaries
parameter_audit.sh          One-command parameter audit entry point
parameter_summary.csv       Generated by parameter_audit.sh
```

Large runtime artifacts are intentionally not committed: datasets, checkpoints, training logs, and full output directories.

## Environment

A working MoleculeSTM environment can be reused. To create a minimal environment for the graph branch:

```bash
conda create -n molstm python=3.7
conda activate molstm
conda install -y -c rdkit rdkit=2020.09.1.0
conda install -y -c conda-forge -c pytorch pytorch=1.9.1
conda install -y -c pyg -c conda-forge pyg==2.0.3
pip install pandas scikit-learn tqdm transformers requests matplotlib spacy Levenshtein ogb==1.2.0
```

The SMILES branch also needs the original MoleculeSTM MegaMolBART/Megatron/Apex dependencies. Reusing a known-good MoleculeSTM environment is recommended for that branch.

## Data And Checkpoints

Set the ChemVL processed-data root:

```bash
export CHEMVL_DATA_ROOT=/path/to/chemvl-data
```

Expected MoleculeNet layout:

```text
${CHEMVL_DATA_ROOT}/finetuning_datasets/MPP/classification/bbbp/processed/bbbp_processed_ac.csv
${CHEMVL_DATA_ROOT}/finetuning_datasets/MPP/classification/bace/processed/bace_processed_ac.csv
${CHEMVL_DATA_ROOT}/finetuning_datasets/MPP/regression/esol/processed/esol_processed_ac.csv
```

MoleculeSTM checkpoints are expected in the original layout by default:

```text
data/pretrained_MoleculeSTM/
data/pretrained_MegaMolBART/checkpoints/
MoleculeSTM/bart_vocab.txt
```

Optional explicit overrides:

```bash
export MOLECULESTM_GRAPH_CKPT=/path/to/graph/molecule_model.pth
export MOLECULESTM_SMILES_CKPT=/path/to/smiles/molecule_model.pth
export MEGAMOLBART_INPUT_DIR=/path/to/pretrained_MegaMolBART/checkpoints
```

## Run Experiments

Graph smoke test:

```bash
python -u chemvl_benchmark/scripts/run_batch.py \
  --tables A \
  --tasks bbbp \
  --runseeds 1 \
  --molecule_type Graph \
  --epochs 1 \
  --output_root outputs/chemvl_benchmark_graph_smoke
```

SMILES smoke test:

```bash
python -u chemvl_benchmark/scripts/run_batch.py \
  --tables A \
  --tasks bbbp \
  --runseeds 1 \
  --molecule_type SMILES \
  --epochs 1 \
  --batch_size 8 \
  --output_root outputs/chemvl_benchmark_smiles_smoke
```

Full Table A/B runs:

```bash
bash chemvl_benchmark/scripts/run_graph_ab.sh
bash chemvl_benchmark/scripts/run_smiles_ab.sh
```

Summarize completed runs:

```bash
python chemvl_benchmark/scripts/summarize.py \
  --moleculestm_output outputs/chemvl_benchmark_graph_runs \
  --output_dir outputs/chemvl_benchmark_graph_tables

python chemvl_benchmark/scripts/summarize.py \
  --moleculestm_output outputs/chemvl_benchmark_smiles_runs \
  --output_dir outputs/chemvl_benchmark_smiles_tables
```

## Parameter Audit

Run one command from the repository root:

```bash
conda activate molstm
bash parameter_audit.sh
```

The script writes:

```text
parameter_summary.csv
```

The audit counts `model.parameters()` after applying the same ChemVL fine-tuning mode used by the reported A/B runs. By default it audits Table A `bbbp` (`num_tasks=1`) for both Graph and SMILES branches.

Useful overrides:

```bash
bash parameter_audit.sh --molecule_types Graph
bash parameter_audit.sh --training_modes fine_tuning linear_probing
bash parameter_audit.sh --output outputs/parameter_summary.csv
```

## Reproducibility Notes

- Fixed split files under `chemvl_benchmark/splits/**/*.npz` are committed because data splitting directly affects performance.
- Runtime artifacts such as `data/`, `outputs/`, checkpoints, logs, and caches should not be committed.
- The SMILES results use the RNG fix documented in `chemvl_benchmark/DECISIONS.md`.
- Checkpoint loading is optional for parameter counting because it does not change model size.
